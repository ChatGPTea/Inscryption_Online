# 基于 Inscryption 风格的 Unity2D 联机卡牌游戏

## 项目概述

这是一个使用 Unity 引擎（版本 **2022.3.62f1c1**）开发的 2D 联机卡牌对战游戏，其美术和玩法设计灵感来源于《**Inscryption**》。项目已开源，支持玩家间实时联机对战。游戏通过网络配置文件（`config.json`）实现服务器连接信息的灵活配置。

## 核心特性

*   **实时联机对战**：实现了低延迟的 PvP 卡牌对战功能。
*   **配置驱动部署**：通过修改根目录下的 **`config.json`** 文件，即可指定服务器地址和运行角色，无需重新编译。
*   **客户端/服务端一体化**：同一套代码可根据 **`role`** 参数分别作为客户端或服务端运行。
*   **Unity 网络技术栈**：采用基于 **Unity WebToken** 的网络解决方案构建联机功能。

## 开发环境

*   **Unity 版本**：**2022.3.62f1c1**（**强烈建议**使用此特定版本以避免兼容性问题）。

## 快速开始

### 1. 获取项目

bash

git clone https://github.com/ChatGPTea/Inscryption_PVP.git

### 2. 打开项目
使用 **Unity Hub** 添加项目，并确保编辑器版本为 **2022.3.62f1c1**。

### 3. 配置网络连接
运行游戏前，**必须**根据您的网络环境配置 `config.json` 文件。该文件位于项目根目录。

配置文件格式与参数说明如下：

json

{

"role": 1,

"httpserver": "http://106.54.11.106:5000/api/
",

"gameserver": "106.54.11.106",

"port": 7777

}
*   **`role`** (整数)：**核心参数**，决定程序运行模式。
    *   **`1`**：作为**客户端**运行，将尝试连接到 `gameserver` 和 `port` 指定的游戏服务器。
    *   **`0`**：作为**服务端**运行，将在本机启动服务并监听 `port` 端口。
*   **`httpserver`** (字符串)：HTTP API 服务器地址，用于处理大厅、匹配、数据统计等非实时请求。
*   **`gameserver`** (字符串)：实时游戏服务器的 IP 地址或域名，用于建立主要的 P2P 或房间制对战连接。
*   **`port`** (整数)：实时游戏服务器监听的端口号。

### 4. 运行游戏
*   **作为服务端运行**：
    1.  将 `config.json` 中的 `"role"` 值修改为 **`0`**。
    2.  在Unity编辑器中点击运行。日志中应显示服务端已在 `[gameserver]:[port]`（如 `106.54.11.106:7777`）上成功启动。

*   **作为客户端运行**：
    1.  确保有一个可用的服务端正在运行，并获取其 IP 地址和端口。
    2.  修改 `config.json`：
        *   `"role"`：设为 **`1`**。
        *   `"httpserver"`：修改为目标服务端对应的 HTTP API 地址。
        *   `"gameserver"` 与 `"port"`：修改为目标服务端的真实 IP 和端口。
    3.  在Unity编辑器中点击运行。客户端将自动尝试连接配置的服务端并进入游戏大厅。

## 重要注意事项

1.  **防火墙与端口**：在局域网或公网部署自建服务端时，请确保在服务器防火墙中开放 `config.json` 中 `port` 指定的端口（默认 `7777`）。
2.  **IP 地址替换**：示例配置文件中的 `106.54.11.106` 仅为占位地址。实际部署时，**必须**将其替换为您服务器的**真实内网 IP** 或**公网 IP**。
3.  **Unity 版本**：**务必**使用指定的 Unity 版本（2022.3.62f1c1）打开项目，不同版本可能导致编译错误或行为异常。
